测试路径解析逻辑：

context-path: /api
Controller mapping: /mock-server/{projectCode}/**
请求URL: /api/mock-server/test/login

期望的解析结果：
- projectCode: test
- path: /login

实际解析过程：
1. fullPath = /api/mock-server/test/login
2. contextPath = /api
3. pathWithoutContext = /mock-server/test/login
4. pathPrefix = /mock-server/test/
5. 检查 pathWithoutContext.startsWith(pathPrefix) -> true
6. path = pathWithoutContext.substring(pathPrefix.length() - 1)
7. path = /mock-server/test/login.substring(19) -> /login

这个逻辑看起来是正确的！
