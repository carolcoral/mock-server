# 使用说明

本系统是一个基于 Spring Boot + Vue 3 的 API 接口模拟服务器，支持自定义接口配置、多项目管理和权限控制。

## 接口调用说明

### 1. 创建项目

登录系统后，进入"项目管理"页面，点击"创建项目"按钮，填写以下信息：

- **项目名称**：项目的中文名称
- **项目编码**：项目的唯一标识，用于接口路径，只能包含字母、数字、下划线和连字符
- **项目描述**：项目的说明信息

### 2. 创建接口

进入"接口管理"页面，点击"创建接口"按钮，配置接口信息：

- **接口名称**：接口的中文名称
- **接口路径**：接口的访问路径，例如 `/api/user/login`
- **请求方法**：支持 GET、POST、PUT、DELETE、PATCH
- **请求类型**：支持 HTTP 和 WebSocket
- **是否启用**：控制接口是否可用
- **响应延迟**：设置接口的响应延迟时间（毫秒）

### 3. 配置接口响应

为接口添加响应配置：

#### 3.1 添加响应

在接口详情页面，点击"添加响应"，填写以下信息：

- **状态码**：HTTP 状态码，例如 200、404、500
- **响应体**：返回的数据内容，支持 JSON、文本等格式
- **响应头**：自定义响应头信息，JSON 格式
- **是否启用**：控制该响应是否可用
- **是否激活**：设置为激活状态，作为默认返回
- **权重**：当启用随机返回时，权重决定返回概率

#### 3.2 条件响应

在响应中设置条件表达式，根据请求参数动态返回不同响应：

```javascript
// 条件表达式示例
$.userId == '123'        // 请求体中 userId 等于 123
$.status == 'active'     // 请求体中 status 等于 active
$.amount > 100           // 请求体中 amount 大于 100
```

**使用场景示例：**
- 当 `userId == '123'` 时返回成功响应
- 当 `status == 'inactive'` 时返回禁用响应
- 当 `amount > 10000` 时返回额度不足响应

#### 3.3 随机响应

启用接口的"随机返回"功能后，系统会根据响应权重随机选择一个激活状态的响应返回：

| 状态码 | 权重 | 概率 |
|--------|------|------|
| 200    | 80   | 80%  |
| 500    | 20   | 20%  |

### 4. 调用 Mock 接口

#### 4.1 HTTP 接口调用

```bash
# 基本格式
curl -X {METHOD} http://localhost:8080/api/mock-server/{projectCode}/{path}

# 示例
curl -X GET http://localhost:8080/api/mock-server/myproject/api/user/list
curl -X POST http://localhost:8080/api/mock-server/myproject/api/user/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"123456"}'
```

#### 4.2 WebSocket 接口调用

```javascript
// 建立 WebSocket 连接
const ws = new WebSocket('ws://localhost:8080/api/ws/mock-server/{projectCode}/{path}')

// 监听消息
ws.onmessage = (event) => {
  console.log('收到消息:', event.data)
}

// 发送消息
ws.send('{"action":"getData"}')
```

### 5. 接口响应示例

#### 5.1 返回 JSON 数据

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "name": "张三",
    "age": 25
  }
}
```

#### 5.2 返回错误响应

```json
{
  "code": 404,
  "message": "用户不存在",
  "timestamp": 1640995200000
}
```

#### 5.3 返回列表数据

```json
{
  "code": 200,
  "message": "success",
  "data": [
    {"id": 1, "name": "张三"},
    {"id": 2, "name": "李四"}
  ],
  "total": 2

```

## 系统功能说明

### 统计信息

首页展示以下统计数据：

- **项目总数**：系统中的项目数量
- **接口总数**：所有项目下的接口数量
- **用户总数**：系统中的用户数量
- **今日请求**：当天内所有项目的所有自定义接口的请求次数
- **历史请求**：截至目前所有项目所有接口的总请求次数

### 系统公告

管理员可以在"系统设置"中创建和管理公告，公告内容支持 Markdown 格式。首页会显示当前启用的公告。

### 用户权限

系统支持三种用户角色：

- **ADMIN（管理员）**：拥有所有权限，可以管理项目、接口、用户和系统设置
- **MEMBER（成员）**：可以查看和编辑自己有权限的项目和接口
- **GUEST（访客）**：只能查看公开的项目和接口

## 常见问题

### Q: 如何设置接口的响应延迟？

A: 在创建或编辑接口时，设置"响应延迟"字段，单位为毫秒。例如设置 1000 表示延迟 1 秒返回响应。

### Q: 如何根据请求参数返回不同的响应？

A: 在响应配置中设置"条件表达式"，例如 `$.userId == '123'`，当请求体中的 userId 等于 123 时返回该响应。

### Q: 随机响应是如何工作的？

A: 启用接口的"随机返回"功能后，系统会根据所有激活响应的权重比例随机选择一个返回。权重越高，被选中的概率越大。

### Q: 如何测试 WebSocket 接口？

A: 使用浏览器开发者工具的 WebSocket 面板，或使用 Postman、Insomnia 等 API 测试工具的 WebSocket 功能。
